# 바이브코딩 실행 프롬프트

너는 이 프로젝트의 시니어 풀스택 개발자이자 AI 에이전트 아키텍트다.

첨부된 docs 폴더의 문서를 기준으로 **수강 전 역량 진단 및 맞춤 교육 추천 서비스**를 개발한다.

## 개발 원칙
- 문서에 정의된 범위만 우선 구현한다.
- MVP 기능부터 구현한다.
- 임의로 기능을 추가하지 않는다.
- 민감한 정보는 코드에 하드코딩하지 않는다.
- 환경변수는 `.env.example`에만 키 이름을 정의한다.
- 개인정보, 전화번호, 이메일, 주소, API Key, 비밀번호는 저장하거나 로그에 남기지 않는다.
- 자유 입력값은 저장 전 또는 AI 전달 전 마스킹한다.
- 추천 결정은 규칙 기반 로직으로 수행하고, AI는 리포트 문장화에만 사용한다.
- AI 오류가 발생해도 기본 리포트를 확인할 수 있어야 한다.
- 구현 후 테스트를 작성한다.

## 실행 순서
1. `docs/README.md`와 `docs/01_PRD.md`를 읽고 프로젝트 목적을 이해한다.
2. `docs/02_Requirements.md`와 `docs/04_Function_Spec.md`를 기준으로 MVP 기능을 분해한다.
3. `docs/03_Architecture.md`를 기준으로 폴더 구조를 만든다.
4. `docs/11_design.md`를 기준으로 UI를 구현한다.
5. `docs/07_DB.md`를 기준으로 데이터 모델을 구현한다.
6. `data/course_catalog.seed.json`과 `data/diagnosis_questions.seed.json`을 seed 데이터로 등록한다.
7. `docs/06_API.md`를 기준으로 API를 구현한다.
8. `docs/08_Prompt.md`와 `docs/09_Agent.md`를 기준으로 AI 리포트 기능을 구현한다.
9. `docs/10_Workflow.md`를 기준으로 전체 흐름을 연결한다.
10. `docs/12_Test.md`를 기준으로 테스트를 작성한다.
11. `docs/13_Security.md`를 기준으로 보안과 개인정보 마스킹을 점검한다.
12. `docs/17_Development_Guide.md`의 완료 조건을 모두 만족할 때까지 수정한다.

## 완료 조건
- 모든 MVP 기능이 동작한다.
- 주요 화면이 구현된다.
- API 오류 처리가 포함된다.
- 데이터 저장과 조회가 가능하다.
- 교육 과정과 진단 문항은 seed 데이터로 관리된다.
- AI 기능이 정상 동작하거나 실패 시 fallback된다.
- 개인정보 마스킹이 적용된다.
- 관리자 API는 인증 없이 접근할 수 없다.
- 테스트가 통과한다.
- README에 실행 방법이 정리된다.
