# Security

## 보안 원칙
- 개인정보 최소 수집
- 자유 입력값 마스킹
- AI 전달 데이터 최소화
- 관리자 권한 분리
- API Key 하드코딩 금지
- 민감정보 로그 저장 금지

## 개인정보 처리
- 진단 서비스는 이름, 전화번호, 이메일, 주소를 입력받지 않는다.
- 진단 자유 입력값은 저장 및 AI 전달 전에 개인정보 마스킹을 적용한다.
- 상담 신청 기능은 회신 목적에 필요한 이름, 소속, 연락처, 이메일(선택), 문의 내용을 저장한다.
- 운영 배포 시 상담 신청 폼에는 수집 목적, 보존 기간, 파기 기준, 문의처를 고지한다.

## 마스킹 규칙
| 유형 | 처리 |
|---|---|
| 이메일 | [email_removed] |
| 전화번호 | [phone_removed] |
| 주소 | [address_removed] |
| 고유식별번호 | 저장 차단 |
| API Key | 저장 차단 |

## 관리자 보안
- 관리자 API는 인증 필요
- 세션 만료 적용
- 권한 없는 접근은 401/403 반환
- 관리자 메모에는 민감정보 입력 금지 안내
