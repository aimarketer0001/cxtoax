# Requirements

## 기능 요구사항
| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-001 | 사용자는 진단 목적과 개인정보 비저장 안내를 확인한다. | Must |
| FR-002 | 사용자는 직무, 관심 분야, 교육 목적, 원하는 결과물을 입력한다. | Must |
| FR-003 | 사용자는 20개 문항에 응답한다. | Must |
| FR-004 | 시스템은 영역별 점수와 총점을 계산한다. | Must |
| FR-005 | 시스템은 자동 보류 조건을 판단한다. | Must |
| FR-006 | 시스템은 수강자 유형을 분류한다. | Must |
| FR-007 | 시스템은 교육 과정 1~3순위를 추천한다. | Must |
| FR-008 | 시스템은 기본 리포트를 생성한다. | Must |
| FR-009 | 시스템은 AI 리포트 생성을 시도하고 실패 시 fallback한다. | Should |
| FR-010 | 관리자는 응답자 목록과 통계를 확인한다. | Must |

## 비기능 요구사항
| ID | 요구사항 |
|---|---|
| NFR-001 | 결과 계산은 AI 호출 없이도 동작해야 한다. |
| NFR-002 | 사용자는 모바일에서도 진단을 완료할 수 있어야 한다. |
| NFR-003 | 추천 로직은 테스트 가능한 순수 함수로 분리한다. |
| NFR-004 | 관리자 API는 인증이 필요하다. |
| NFR-005 | 자유 입력값은 개인정보 마스킹을 거친다. |

## 권한 요구사항
| 역할 | 권한 |
|---|---|
| Guest | 진단 응답, 결과 확인 |
| Admin | 응답 목록, 상세 리포트, 통계 조회, 과정 관리 |
| System | 점수 계산, 유형 분류, 추천 산출 |
| AI | 익명화된 결과 기반 리포트 문장화 |

## 예외 조건
| 상황 | 처리 |
|---|---|
| 필수 문항 누락 | 해당 문항으로 이동하고 안내 |
| AI API 실패 | 기본 템플릿 리포트 사용 |
| 추천 결과 없음 | 상담 권장 또는 기초 과정 안내 |
| 관리자 인증 실패 | 로그인 화면 이동 |
| DB 저장 실패 | 재시도 안내 |
