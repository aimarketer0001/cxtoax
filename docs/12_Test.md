# Test 계획서

## 테스트 범위
| 영역 | 테스트 |
|---|---|
| 진단 폼 | 문항 표시, 선택, validation |
| 점수 계산 | 영역별 점수, 총점 |
| 유형 분류 | 점수 구간, 보류 조건 |
| 추천 로직 | 직무/목적별 추천 |
| AI 리포트 | 성공, 실패, JSON 오류 |
| 관리자 | 로그인, 목록, 상세, 통계 |
| 보안 | 개인정보 마스킹, 권한 제어 |

## 핵심 테스트 케이스
| 케이스 | 조건 | 기대 결과 |
|---|---|---|
| 업무 적용형 | 영업+CRM+업무효율 | CX-B-01 또는 CX-B-04 추천 |
| 상담/CS형 | CS+FAQ+응대 | CX-B-02 추천 |
| 입문형 | AI 이해도 낮음 | CX-E-01/CX-E-02/CX-D-01 추천 |
| 전문가형 | 실행력 높고 HRD/강사 목적 | CX-E-03 추천 |
| 보류형 | 실행력 10점 미만 | 상담 권장 또는 보류 |

## AI 테스트
- JSON Schema 준수
- 추천 교육 ID 변경 금지
- 개인정보 포함 여부 검사
- 실패 시 기본 리포트 표시

## 완료 기준
- 핵심 단위 테스트 통과
- 주요 E2E 시나리오 통과
- AI fallback 테스트 통과
- 관리자 권한 테스트 통과
